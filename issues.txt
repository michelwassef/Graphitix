# Open Issues (Reviewed 2026-06-01)
# Ordered by descending priority. Closed and stale items were removed.

[open][P0][2026-06-01] Restore-draw scheduling remains fragmented and brittle across components: resize cache invalidation policy, reason-string suppression, and draw-option merge precedence are not centralized, so user-initiated redraws can still be suppressed or over-expanded after reopen. Replace with one shared scheduler/suppression + merge policy contract. (merged from 2026-05-31 issues 89, 90, 91, 95)

[open][P0][2026-06-01] Stats restore authority is still inconsistent between payload apply and activation/draw suppression flows; restored stats metadata can diverge from runtime signature inputs and be invalidated later. Standardize one payload-time stats hydration contract across components before context priming. (merged from 2026-05-22 issues 43, 44)

[open][P0][2026-06-01] Scatter/box payload serialization and control state acquisition still depend on live DOM in critical paths instead of state-first snapshots, keeping inactive-tab/headless flows fragile. Introduce a shared state-authoritative accessor contract. (from 2026-05-19 issue 32)

[open][P1][2026-06-01] Render-cache behavior lacks a shared computation/cache metadata contract and still duplicates canvas bitmap hydration logic. Heavy tabs can reopen visually but miss warm computational caches, then stall on first interaction. Consolidate capture/serialize/restore/validate for computation and bitmap caches. (merged from 2026-05-23 issue 45 and 2026-05-31 issue 92)

[open][P1][2026-06-01] Viewport-fit behavior remains inconsistent across components (2D/3D sequencing, lock semantics, preserve/fill modes), producing cross-component resize/render drift and avoidable letterboxing divergences. Define one shared viewport policy with explicit modes and phase timing. (merged from 2026-05-18 issues 25, 27, 29 and 2026-05-30 issue 85)

[open][P1][2026-06-01] Bottom-axis reserve/rotation behavior is still partly duplicated and contract-incomplete (`computeBottomLayout` always bakes axis-title reserve). Add explicit shared axis-title/reserve options and component-wide rotation hysteresis adoption. (merged from 2026-05-18 issues 26, 30)

[open][P1][2026-06-01] Shared chart-style/theming helpers remain fragmented: resize scale policy is global-only, `applySvgDefaults` still mixes style with interactive font-control binding, and line/scatter SVG theme logic is duplicated across draw paths. Split responsibilities and centralize theme/style application helpers. (merged from 2026-05-18 issues 22, 23, 24)

[open][P1][2026-06-01] Box color intent is still inferred from color value (`isBoxThemeNeutralColorToken`) rather than explicit user-intent flags, so value collisions with defaults can still remap deliberate choices. Persist explicit per-color intent/state. (from 2026-05-30 issue 86)

[open][P1][2026-06-01] SVG frame publication lifecycle is still not standardized (build->validate token->publish->commit). Some paths can expose partial/stale intermediate frames. Introduce a shared frame publication contract with cancellation cleanup. (from 2026-05-26 issue 71)
