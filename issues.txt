[open][P1][2026-06-04] Long graph/stat render work lacks a shared cooperative cancellation and responsiveness contract. ROOT CAUSE: components mix passive loading overlays, per-component draw tokens, partial async-scope stale-result guards, worker offload for only some phases, and long synchronous SVG/canvas/data loops. A "Stop drawing" button alone would not reliably work because the main thread cannot process the click while inside a blocking loop, and current Shared.Workers.runTask can ignore stale results but cannot cancel/terminate a specific in-flight task. BIGGER REFACTOR: introduce a shared tab-scoped render/job controller that owns render tokens, AbortSignal-style cancellation, overlay stop/retry UI, worker task IDs with cancel/terminate support, chunk/yield helpers for large main-thread loops, frame-publication cleanup on cancel, and idle/busy state for save/recovery. Normalize all components to use it for graph drawing, stats, imports, analysis, and restore redraws, preserving true tab isolation. Add real-browser responsiveness/performance guards with large datasets that assert cancel is clickable, stale work cannot publish, retry redraws from the same payload, and unrelated tabs continue unaffected. Related but broader than the 2026-06-01 SVG frame publication lifecycle issue.

[open][P1][2026-06-04] Venn/Pie/Survival still do not participate in the shared render loading-overlay contract used by scatter/box/pca/line/heatmap/hist/roc/surface. Normalize these components onto the same render-job controller, with tab-scoped cancelCurrentDraw hooks and large-dataset responsiveness guards, so graph cancellation truly covers every component homogeneously.

[open][P1][2026-06-05] Archive/recovery render-cache restores lack a shared "live interaction hydration" contract. ROOT CAUSE: domControls accepts archive-backed render caches as authoritative, applies payload with skipDraw/skipInitialDraw, restores serialized graph fragments, and suppresses follow-up draws; but serialized SVG/DOM nodes do not preserve per-node JS listeners, closure state, WeakMap registrations, or component-private render models. Components therefore reopen with visually correct but behaviorally stale graphs: box restored plot nodes can miss point/shape/axis/font toolbar bindings, while heatmap palette edits can update state but be swallowed by restore draw suppression or operate without a rebuilt render model. BIGGER REFACTOR: define one restore contract for graph interactivity: either (A) render cache is visual preview only and every activated reopened tab immediately performs a real draw from restored payload before accepting graph edits, or (B) each component exposes an explicit hydrateRestoredGraphInteractions/cacheHydrate hook that re-registers fontControls, axisControls, symbol/shape handlers, 3D/drag handlers, render models, and palette/legend bindings after restore. The contract must clear post-restore suppression on user edit, validate that restored graphs are "live", and include real-browser reopen/recovery guards for toolbar visibility plus style/color changes across scatter, box, heatmap, line, and other SVG-heavy components. Scatter currently behaves closer to the intended model because activation reapplies payload-derived visual/runtime state and its graph-only cache is less dependent on serialized stats/interaction DOM; use it as the normalization reference but do not assume its cache restore is fully certified without these guards.

[open][P1][2026-06-05] User style/control edits are not normalized onto a synchronous payload write-through contract, so recovery can persist stale graph styling. ROOT CAUSE: many toolbar callbacks mutate component-local state and redraw (e.g. box point/shape style maps, heatmap palette/value-scale state) while relying on global dirty tracking plus later live `getPayload()` reconciliation; manual `.graph` save is save-like and forces that reconciliation, but recovery/autosave use lean lifecycle snapshot policies that may preserve cached `tab.payload` when the tab appears already flushed or clean. BIGGER REFACTOR: define one component mutation API for payload-affecting visual edits that atomically updates local state, updates or invalidates `tab.payload`, marks the owning tab dirty, clears stale render caches, and schedules recovery from the new payload signature. Replace ad hoc state-only callbacks in symbol/color/palette/axis/font toolbars with that API, and add real-browser recovery guards that edit box symbols and heatmap colors, wait for recovery, reload, accept recovery, and assert payload + rendered color/shape parity. Related to the state-first snapshot accessor P0 and archive/recovery live interaction hydration issue, but distinct because the data loss happens before restore: the recovery archive can be stale even when reopen of an explicitly saved file is correct.
