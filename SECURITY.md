# Security Policy

## Supported Versions

Security updates are provided for the latest `main` branch and the most recent tagged release.

## Reporting a Vulnerability

Please do not open public GitHub issues for security-sensitive reports.

1. Email the maintainer at the address listed in repository metadata.
2. Include: affected version/commit, reproduction steps, impact, and suggested mitigation if available.
3. Expect an initial response within 7 days.

## Disclosure Process

- We validate and triage reports privately.
- A fix is prepared and tested before coordinated disclosure.
- Release notes will acknowledge the issue and mitigation details when publication is safe.
